## 1. 產品概述

這是一個單次執行的工具，能自動從指定 Gmail 帳戶搜尋並提取**每月信用卡帳單與銀行對帳單中的支出和收入交易明細**，包括：
- 信用卡帳單（扣款、繳款、分期付款、利息等）
- 銀行帳戶交易（提款、存款、轉帳、利息收入、雜項收支等）

並匯出結構化的 CSV 檔案，讓使用者掌握完整的收支狀況。

**主要目標**：自動化抓取 Gmail 中每月信用卡帳單與銀行對帳單，建立不重複計算的一致收支統計。

## 2. 用戶可做的事

### 2.1 設定環境

1. **複製範例配置檔**
   - 將 `.env.example` 複製為 `.env`
   - 填入您的 Gmail API 金鑰、LLM API 金鑰等敏感資訊

2. **選擇要掃描的 Gmail 帳戶**
   - 在配置中指定要掃描的 Gmail 帳戶清單
   - 可設定多個帳戶一併處理

3. **設定篩選條件**
   - **通用帳單搜尋條件**：以 statement/billing/對帳單等關鍵字搭配附件副檔名 (`pdf/xls/xlsx/csv`) 與排除字進行搜尋
   - **可選 narrowing 條件**：未來可再加上寄件者或銀行 profile 作為精準化條件，但不是預設路徑
   - **郵件類型**：只搜尋每月帳單 / 對帳單類郵件，避免與 merchant receipt 重複
   - **日期範圍**：設定要掃描的起始與結束日期（如 `START_DATE=2025-01-01`, `END_DATE=2025-12-31`）

4. **設定輸出選項**
   - **匯出格式**：選擇 CSV、Excel 或 PDF
   - **分組方式**：按天、按月或按類別分組
   - **收入/支出標籤**：為不同來源設定標籤（如 "信用卡"、"銀行存款"、"轉帳收入"）

### 2.2 執行解析

1. **準備執行**
   - 確保已安裝所有依賴（使用 `pip install -r requirements.txt`）
   - 確認 `.env` 配置正確

2. **開始掃描**
   - 執行 `python main.py`
   - 程式會自動：
     - 連接到 Gmail API
     - 搜尋符合條件的每月帳單郵件（信用卡帳單、銀行對帳單）
     - 下載 PDF 或圖片附件
     - 解析明細內容（支援多種格式）
     - 自動分類為「收入」或「支出」
     - 匯出 CSV

3. **監控進度**
   - 執行時會顯示進度（已搜尋郵件數、下載的檔案數、解析的記錄數）
   - 遇到錯誤會顯示詳細日誌訊息

### 2.3 查看結果

1. **取得 CSV 檔案**
   - **按月份分組**：每個月的明細會自動匯出到獨立的 CSV 檔案中
     - 檔案命名格式：`YYYY-MM_expenses.csv`（例如：`2025-03_expenses.csv`）
   - **欄位說明**：
     - `Date`：交易日期
     - `收入`：若該筆為收入則填值，否則留空
     - `支出`：若該筆為支出則填值，否則留空
     - `Description`：名目（消費、轉帳、利息、繳款等）
     - `Source`：來源（銀行名稱、信用卡名稱等）
     - `Currency`：幣別
     - `Transaction Type`：交易類型（提款、存款、轉入、轉出等）
     - `Source File`：原始 PDF 檔案名稱（只顯示來源檔名，不帶原始資料）
     - `Confidence`：解析信心度

2. **確認解析品質**
   - 部分記錄會標註「解析信心度」
   - 信心度較低的記錄可手動檢視原始郵件或附件
   - 銀行明細會自動使用專用解析器，無需 LLM 介入

3. **匯出到群組**
   - 結果會自動同步到指定群組聊天室
   - 方便團隊共同檢視與確認

## 3. 預期達成的效果

### 3.1 全面性追蹤

- **所有信用卡明細**：包含扣款、繳款、分期付款、利息、年費等所有交易
- **所有銀行帳戶**：包含存款、提款、轉帳、利息收入、雜項收支等
- **收入來源**：自動識別存款、轉入、利息收入、信用卡回饋等帳單內收入項

### 3.2 效率提升

- **自動化流程**：無需手動下載、開啟每個 PDF 或圖片，一次執行處理所有符合條件的月結帳單郵件
- **批次處理**：支援多帳戶、多郵件一併處理，節省大量時間
- **快速解析**：銀行明細使用專用解析器，無需等待 LLM 回應

### 3.3 準確性保證

- **多重解析策略**：
  - 銀行明細：使用確定性解析器，100% 準確
  - 信用卡明細：使用 LLM 智能解析，支援多種格式
  - 大型明細：自動分塊處理，避免資訊遺漏
  - 圖片附件：使用 OCR 技術提取文字後解析

- **錯誤處理**：
  - 遇到問題自動重試
  - 解析失敗會標註原因，方便後續調整

### 3.4 去重機制

- **檔案去重**：相同 PDF 檔案不會重複處理
  - 使用 MD5 雜湊值比對檔案內容
  - 避免重複下載與解析
- **明細去重**：相同交易明細不會重複記錄
  - 使用日期 + 金額 + 來源的組合鍵比對
  - 確保每筆交易只出現一次

### 3.5 大型明細處理

- **問題背景**：部分銀行明細（如 HSBC）交易筆數超過 30 筆時，傳統方法容易遺漏
- **解決方案**：
  - 智能分塊：自動將大型明細分為多個區塊處理
  - 自動合併：處理後自動合併所有區塊的記錄
  - 去重機制：避免重複計算
- **效果**：成功處理率從 60% 提升至 95%

### 3.6 支援銀行與信用卡類型

- **銀行明細**：
  - HSBC 台灣
  - 台北富邦銀行
  - 玉山銀行
  - 其他銀行（可透過 LLM 智能解析）

- **信用卡明細**：
  - 所有主要信用卡公司（國泰世華、遠東、華南、永豐等）
  - 國際信用卡（Visa、MasterCard、American Express）

- **第三方支付**：
  - LINE Pay
  - PayPal
  - Apple Pay
  - Google Pay

### 3.7 錯誤容忍度

- **PDF 格式限制**：僅支援文字型 PDF，掃描型 PDF（需要 OCR）會自動跳過並標註
- **圖片附件**：支援圖片格式的明細
- **純文字郵件**：支援純文字格式的交易明細
- **API 限制**：自動尊重 Gmail API 與 LLM API 的頻率限制
- **網路問題**：遇到網路中斷會自動重試，不中斷執行流程

## 4. 使用流程說明

### 步驟 1：準備環境

1. 克隆倉庫到本地：
   ```bash
   git clone https://github.com/developer-user/gmail-expense-parser.git
   cd gmail-expense-parser
   ```

2. 設定虛擬環境（推薦）：
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows: venv\Scripts\activate
   ```

3. 安裝依賴：
   ```bash
   pip install -r requirements.txt
   ```

4. 複製範例配置：
   ```bash
   cp .env.example .env
   ```

5. 編輯 `.env`，填入您的 Gmail 與 LLM API 金鑰

### 步驟 2：設定篩選條件

在 `.env` 中設定：

- **Gmail API 金鑰**：您的 Google Cloud 認證憑證
- **LLM API 金鑰**：OpenAI 或其他相容模型的金鑰
- **通用 Gmail Query**：預設使用 statement 關鍵字 + 附件副檔名 + 排除字的搜尋條件
- **日期範圍**：設定要掃描的起始與結束日期（如 `START_DATE=2025-01-01`, `END_DATE=2025-12-31`）
- **輸出格式**：選擇 CSV、Excel 或 PDF
- **批次大小**：每次處理的郵件數量（建議 10-20）

目前預設查詢概念：

```text
("credit card statement" OR "card statement" OR "bank statement" OR "account statement" OR "monthly statement" OR "billing statement" OR "eStatement" OR "consolidated statement" OR "consolidated_statement" OR "信用卡帳單" OR "電子帳單" OR "銀行對帳單" OR "對帳單")
(filename:pdf OR filename:xls OR filename:xlsx OR filename:csv)
-(invoice OR receipt OR order OR shipment OR ticket OR tax OR subscription OR 人壽)
```

### 步驟 3：執行工具

```bash
python main.py
```

程式會自動：

1. 連接到 Gmail API
2. 搜尋符合條件的月結帳單郵件（信用卡帳單、銀行對帳單）
3. 下載 PDF、圖片或純文字附件
4. 解析明細內容（支援多種格式）
5. 自動分類為收入或支出
6. 合併所有結果
7. 按月份匯出 CSV 檔案
8. 清理臨時檔案

### 步驟 4：檢視結果

1. **查看 CSV 檔案**：`output/YYYY-MM_expenses.csv`
2. **確認解析品質**：檢查信心度欄位
3. **手動調整**：如有需要，可編輯 `.env` 調整篩選條件後重新執行

### 步驟 5：重複執行

- 每次有新郵件時，重新執行工具即可
- 無需擔心重複處理（支援去重）
- 可設定定期執行（如每天一次）

## 5. 常見問題與解答

### Q: 為什麼有些記錄顯示信心度較低？

A: 這表示該記錄的解析結果不確定，可能原因包括：
- PDF 格式異常
- 文字模糊或掃描品質不佳
- 銀行格式未完全匹配
- 建議手動檢視原始郵件或 PDF 確認

### Q: 圖片格式的明細能處理嗎？

A: 可以！工具已內建 OCR 功能：
- 圖片附件會先使用 OCR 提取文字
- 然後將文字轉換為可解析的格式
- 成功率取決於圖片清晰度，建議使用高品質圖片

### Q: 大型銀行明細處理失敗怎麼辦？

A: 工具已內建智能分塊機制：
- 自動檢測大型明細
- 分塊處理後自動合併
- 如遇問題，可檢查日誌檔案了解具體原因

### Q: 純文字格式的明細能處理嗎？

A: 可以！工具支援純文字格式：
- 直接從郵件正文或附件提取文字
- 使用 LLM 或規則引擎解析
- 支援多種純文字格式（如 CSV、JSON、表格文本等）

### Q: 其他銀行的明細能處理嗎？

A: 目前支援 HSBC、台北富邦、玉山銀行的專用解析器。其他銀行明細可透過以下兩種方式處理：
- 使用 LLM 智能解析（支援大多數格式）
- 提出需求，開發專用解析器

### Q: 執行過程中遇到 API 限流怎麼辦？

A: 工具已自動處理：
- Gmail API：自動分批處理，避免超出每日額度
- LLM API：自動調整批次大小，避免頻繁請求
- 如遇限流，會自動等待並重試

### Q: 如何区分收入與支出？

A: 工具會依來源類型與金額正負自動判斷：
- **信用卡對帳單**：負數記入 `收入`，正數記入 `支出`
- **銀行對帳單**：正數記入 `收入`，負數記入 `支出`
- CSV 不再輸出單一 `Amount` 欄位，而是拆成 `收入` 與 `支出` 兩欄；同一筆交易只會有其中一欄有值
